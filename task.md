**3.1.13** **企业访客小程序设计与实现**

企业访客预约系统是一种利用微信小程序平台开发的访客管理解决方案,系统旨在帮助

企业高效地管理访客预约、接待和信息记录，提升企业的服务水平和效率。通过微信小程序

，访客可以方便快捷地进行预约操作，企业可以实时获取访客信息并做好接待准备。系统

通过智能化的预约管理和安全控制， 为企业提供了一种便捷、 安全、 高效的访客管理方式

。

系统共有四个角色分别为管理员、访问人、被访问人以及门岗。其中管理员角色主要负

责人员管理、预约申请审批、发布通知以及相关数据统计，人员管理包含对员工的条件查询

、增删改查和相关批量操作。被访问人主要负责对其预约审批的管理和对到访信息查看。访

问人主要有预约管理和查看进入须知，访问人的预约管理主要包含预约申请、撤回预约申请

等功能。 门岗主要是对访问人的预约申请结果进行确认并放行。

### 开发与部署方案

**技术栈**
- 后端：Spring Boot 3 + MyBatis+ MySQL + Redis（可选）
- 前端：微信小程序原生
- AI：调用大模型API（DeepSeek / 通义千问）

**微信API使用清单**
| API | 是否需申请 | 用法 |
|-----|-----------|------|
| wx.login() | 需AppID，免费注册 | 登录换JWT |
| wx.scanCode() | 不需申请 | 门岗扫码核验 |
| wx.requestSubscribeMessage | 需申请模板 | 预约推送（课设可跳过） |
| 二维码 | 后端ZXing生成 | 不需微信API |
| 用户信息 | 手动填写表单 | 不走微信授权 |

**部署方案（零成本）**
- 开发阶段：微信开发者工具勾选"不校验合法域名"，后端直接 `http://localhost:8080`
- 演示阶段：用 **natapp** 或 **ngrok** 内网穿透，将本地 Spring Boot 映射为 HTTPS 地址
- 或部署到 Railway / Zeabur 免费云托管

**开发顺序**
1. 搭建后端框架 + 用户登录认证
2. 员工/部门管理（管理员）
3. 预约申请 + 审核流程（核心业务流）
4. 预约记录、二维码、通知
5. 门岗扫码核验
6. AI话术生成（必选）
7. AI风险预警（加分）
